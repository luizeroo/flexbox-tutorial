O que é flexbox e quando utilizar?

O módulo Flexbox é uma recomendação da W3C - é a principal organização de padronização da World Wide Web (WWW) - desde outubro de 2017 e atualmente é utilizado em maior parte dos navegadores modernos, porém, sendo apenas parcialmente usado no IE9 e I10. Sua taxa de uso nos navegadores é de 98% de implementação - enquanto o "CSS Grid" bate os 93%.

Flexbox tem por objetivo fornecer uma forma mais fácil e objetiva de alinhar e distribuir os objetos dentro de um container, sempre levando em consideração que o tamanho pode ser ou não desconhecido. Lucas Henrique de Abreu

Pera aí, então devo usar sempre FlexBox?

FlexBox VS CSS Grid

A maior "rixa" que existe é entre os desenvolvedores é se o FlexBox deve ser ou não substituído pelo CSS Grid, porém:

  • FlexBox faz coisas que o CSS Grid não pode fazer
  • CSS Grid faz coisas que o FlexBox não pode
  • Ambos podem trabalhar em conjunto
  • Há coisas que ambos podem fazer, mas que um faz de uma maneira mais rápida e simples

FlexBox

Claro que podemos usar o flexbox para trabalhar com elementos em duas dimensões, mas flexbox é mais bem utilizado quando estamos trabalhando com elementos em apenas uma dimensão, seja linha ou coluna. Um bom exemplo é quando criamos um menu, seja ele vertical ou horizontal, ideal para alinhar elementos, não importa a direção.

Grid

Claro que podemos usar o Grid para trabalhar com elementos em uma única dimensão, mas ele é ótimo quando estamos trabalhando com elementos em duas dimensões, principalmente para definir a estrutura de uma página, o que seria muito mais "difícil" com FlexBox.

Noções básicas

Flexbox é um módulo inteiro e não apenas uma única propriedade, por isso, envolve um conjunto de propriedades. Essas propriedades normalmente setamos para o elementos pai do contexto que queremos atuar com flexbox. Para isso, usamos a propriedade Display, que define se um container é flexível, inline ou bloqueado (que é o padrão de todo elemento de bloco)

O que são elementos de bloco?

Elementos bloco ocupam todo o espaço horizontal disponível e iniciam uma nova linha no documento. Novos elementos irão começar na próxima linha livre.

  • div (division, ou divisão, tag bloco genérica)
  • h1 até h6 (títulos)
  • p (parágrafo)
  • blockquote (citação em bloco)
  • ul (lista não ordenada)
  • ol (lista ordenada)
  • form (para formulários)

flex-grow

Define a habilidade de um item crescer.

Ao definir 1 pra todos, eles tentarão ter a mesma largura e vão ocupar 100% do container - caso um dos elementos tenha um conteúdo muito largo, tentará respeitar o mesmo.

Justify-content não funciona com flex-grow definido.
